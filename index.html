Überblick zu APIs

Was ist eine API?

  • nicht präzise definiert
  • unterschiedliche Definitionen
  • Begriff verschieden weit gefasst

Literaturhinweise

  • Irresistible APIs, S. 3-10
  • Designing Web APIs. S. 1-26
  • Außerdem zum Gesamtverständnis zu HTTP und REST hilfreich: Irresistible APIs, S. 64-77.
  • Seminarhandbuch "Webbasierte Forschungsapplikationen"

Begriff API

  • Application Programming Interface bzw.  Programmierschnittstelle
  • Ein Interface: Anders als ein User Interface, das menschenzentrierte Interaktion bietet, bietet eine API die Möglichkeit einer Interaktion zwischen zwei Maschinen/Programmen
  • Daher zu beachten bei Design: so wie ein User Interface konzipiert und gestaltet wird, muss auch eine API gestaltet werden

Definitionen

"Programmteil, der von einem Softwaresystem anderen Programmen zur Anbindung an das System zur Verfügung gestellt wird"

Wikipedia

"API stands for Application Programming Interface. It refers to a set of tools, protocols or routines used for building applications. An API informs software how it must interact."

Wikipedia EN

"promoting interoperability between a variety of components and systems"

Erstmalige Nennung des Begriff API und Definition 1968 in "Data Structures and Techniques for Remote Computer Graphics"

Unterscheidung und Kategorisierung

Interne API
Aufteilung von Code in einzelne Komponenten
Steigert Modularität, verringert Komplexität
Service-Oriented Architectures: Gesamtsystem in unabhängige Teil-Systeme zerlegt
Zielgruppe: Man selbst oder Team-Mitglieder
Externe API
Meistens gemeint, wenn über APIs gesprochen wird
Bereitstellung von Funktionen und Daten
Zielgruppe: externe Partner, "Das Internet"

Entwicklung des Begriffs API

Von verbundenen zu entkoppelten Systemen

Ursprünglich

  • Interface zwischen Computersystemen
  • Meist eng verbundene Systeme, bei denen die API nicht auch von externen genutzt werden sollte
  • APIs als Architekturstil für modulare Softwarearchitektur

Heute¹

  • Verwendung des Begriffes API heute (und vor allem im DH Kontext) meistens im Sinne von Web APIs, also Schnittstellen von Websites und Webapplikationen
  • Systeme sind stärker entkoppelt

¹ Seit wann gibt es die ersten Web APIs? Ende 90er: SOAP/XML endpoints. 2006: Twitter und Facook API.

Web APIs

  • REST (Representational State Transfer): Architekturstil / Programmierparadigma
  • HTTP (Hypertext Transfer Protocol): Protokoll für die Kommunikation
  • JSON / XML: Format für Formulierung von response und request

REST - 6 constraints

Uniform Interface
Einheitliche Schnittstelle durch Adressierbarkeit von Ressourcen, Repräsentationen zur Veränderung von Ressourcen, Standardmethoden zur Manipulation von Ressourcen, Hypermedia as the Engine of Application State“ (HATEOAS): Informationen werden dynamisch über Hypermedia gegeben, Client benötigt wenig Wissen über Interaktione
Statelessness
Zwischen zwei Nachrichten werden keine Zustandsinformationen gespeichert
Cacheable
HTTP Caching soll genutzt werden.
Client-Server
Server stellteinen Dienst bereit, der vom Client angefragt werden kann
Layered System
Systeme sollen mehrschichtig aufgebaut werden.
Code on Demand (optional)
Erst im Bedarfsfall Code zur lokalen Ausfürhung an Client übertragen.

Definition im Rahmen dieses Seminars

Eine Programmierschnittstelle stellt Daten offen in einem maschinenlesbaren Format bereit und soll von einer bestimmten Zielgruppe genutzt werden.

Wozu werden Web APIs genutzt?

Grafik: Hunter, Irresistible APIs, 2017

Wozu werden Web APIs genutzt?

  • Ermöglichen Kommunikation zwischen zwei Systemen
  • Dienen dem Austausch und der Weiterverarbeitung von Daten und Inhalten verschiedener Websites / Applikationen

APIs geben:

  • Zugang zu Daten
  • Zugang zu Diensten
  • Grundstein der Softwareentwicklung / Softwarearchitekturmuster

APIs in den Geisteswissenschaften

Besonderheiten von APIs in den Geisteswissenschaften

  • APIs primär, um Daten zu beziehen
  • meist nur zur Datenabfrage
  • deutlich höherer Standardisierungsgrad was Links und Vokabularien betrifft als in anderen Kontexten
  • XML vor JSON
  • RESTful zwar weit verbreitet, aber auch andere Formate dominieren

APIs in den Geisteswissenschaften
